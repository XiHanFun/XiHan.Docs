# 项目简介

## 关于 XiHan

**XiHan（曦寒）** 是一个开源的、企业级的全栈开发解决方案，旨在帮助开发者快速构建安全、稳定、高效的业务系统。

## 设计理念

### 快速启动优先

最小化配置，开箱即用。框架提供合理的默认配置，让您可以在几分钟内启动一个功能完整的应用程序。

### 模块化设计

采用灵活的模块化架构，每个功能都是独立的模块，您可以按需引入所需功能，避免不必要的依赖和性能开销。

### .NET 原生优先

充分利用 .NET 10 的原生功能和性能优势，优先使用平台提供的标准实现，而不是引入第三方替代方案。

### 国际化友好

提供完整的中文文档和注释，同时兼容国际标准和最佳实践，适合国内外项目使用。

### 性能至上

从设计之初就将性能作为首要考虑因素，支持 AOT 编译、异步编程、高性能序列化等特性。

## 适用场景

XiHan 适用于以下场景：

- **企业管理系统** - ERP、CRM、OA 等企业内部管理系统
- **SaaS 平台** - 多租户 SaaS 应用

- **内容管理系统** - CMS、博客、文档管理
- **微服务应用** - 构建微服务架构的基础设施

## 架构设计

### 分层架构

XiHan 采用经典的分层架构：

**表现层（Presentation Layer）**
- Web API 控制器
- SignalR Hub
- 中间件管道

**应用层（Application Layer）**
- 应用服务
- DTO 定义
- 自动映射

**领域层（Domain Layer）**
- 聚合根和实体
- 领域服务
- 领域事件
- 业务规则

**基础设施层（Infrastructure Layer）**
- 数据访问实现
- 外部服务集成
- 缓存实现
- 消息队列

### 模块依赖

```
表现层
  ↓
应用层
  ↓
领域层
  ↓
基础设施层
```

每一层只依赖下层，保持清晰的依赖关系。

## 技术选型原则

1. **成熟稳定** - 选择经过实践检验的技术栈
2. **性能优先** - 优先考虑性能表现
3. **生态丰富** - 选择社区活跃、生态完善的技术
4. **持续演进** - 跟进最新技术发展，及时升级
5. **国内友好** - 考虑国内网络环境和使用习惯
